# Text Processor (Local DAGL Workflow)

A simple text processing pipeline demonstrating a **local-only** unum workflow
with type-checked functions using `@input`/`@output` annotations.

## Workflow

```
Tokenize → Analyze → Report
```

Each function has type annotations in its docstring:

- **Tokenize**: `{text, language}` → `{words, count, language}`
- **Analyze**: `{words, count, language}` → `{topWords, uniqueCount, totalCount}`
- **Report**: `{topWords, uniqueCount, totalCount}` → `{report, stats}`

The output types of each function match the input types of the next function,
so type validation passes cleanly.

## Source files (what you write)

```
text-processor-dagl/
├── workflow.dag           # DAGL workflow definition
├── Tokenize/app.py       # Function code with @input/@output annotations
├── Analyze/app.py        # Function code with @input/@output annotations
├── Report/app.py         # Function code with @input/@output annotations
└── input.json            # Sample input for testing
```

## Step-by-step

### 1. Compile

Generates `unum_config.json` for each function, `unum-template.yaml`, and
validates types across edges.

```bash
cd text-processor-dagl
unum-cli compile --platform dagl --workflow workflow.dag
```

With strict type checking (fails on type mismatch):

```bash
unum-cli compile --platform dagl --workflow workflow.dag --strict-types
```

**Generated files after compile:**

```
├── Tokenize/unum_config.json   (generated)
├── Analyze/unum_config.json    (generated)
├── Report/unum_config.json     (generated)
└── unum-template.yaml          (generated)
```

**Generated files after template:**

```
└── template.yaml               (generated - SAM format)
```

### 2. Generate SAM template

Converts `unum-template.yaml` (generated by compile) into `template.yaml`
(AWS SAM format).

```bash
unum-cli template
```

### 3. Build

Packages functions with the unum runtime (copies `main.py`, `unum.py`, etc.
into each function directory via SAM build).

```bash
unum-cli build
```

Or combine steps 2+3 in one command:

```bash
unum-cli build -g
```

### 4. Deploy

Deploys the SAM stack to AWS.

```bash
unum-cli deploy
```

### 5. Invoke (optional)

How to run and get the final response:

```bash
unum-cli invoke -f input.json --wait
```
